### Apps/Create.cshtml — ساخت ابزار جدید
فرم چند مرحله‌ای (Wizard):

**مرحله ۱: اطلاعات پایه**
- عنوان
- توضیح کوتاه
- توضیح کامل (Markdown Editor)
- دسته‌بندی
- تگ‌ها

**مرحله ۲: تنظیمات AI**
- انتخاب نوع خروجی (OutputType)
  - وقتی کاربر یکی را انتخاب می‌کند، مدل‌های سازگار نمایش داده می‌شوند
- انتخاب مدل AI از لیست
- هزینه اعتباری (با راهنمایی محاسبه قیمت)

**مرحله ۳: پرامپت**
- System Context (اختیاری) — دستور سیستمی
- پرامپت اصلی
  - Textarea با نمایش متغیرها به صورت {variable_name}
  - پرامپت بلافاصله رمزنگاری می‌شود
- راهنمای استفاده از متغیرها

**مرحله ۴: فیلدهای ورودی**
- این مرحله مستقیم به صفحه Fields/Manage هدایت می‌کند

### Apps/Edit.cshtml — ویرایش ابزار
- فقط برای ابزارهای Draft یا Suspended
- اگر ابزار Active باشد، فقط عنوان و توضیح قابل ویرایش است (نه پرامپت و AI)
- برای ویرایش پرامپت، باید به Draft برگردد

---

### Apps/Fields.cshtml — مدیریت فیلدهای ورودی (Form Builder)
قلب تجربه سازنده — اینجاست که فرم کاربر ساخته می‌شود.

**رابط کاربری:**
- سمت چپ: پنل اضافه کردن فیلد
- سمت راست: پیش‌نمایش زنده فرم

**برای هر فیلد:**
- Label (نام نمایشی)
- Name (نام متغیر که باید با {name} در پرامپت مطابقت داشته باشد)
- نوع فیلد:
  - Text — ورودی یک خطی
  - Textarea — ورودی چند خطی
  - Select — لیست انتخابی (مقادیر را وارد می‌کند)
  - Number — عدد
  - Checkbox — بله/خیر
  - FileUpload — برای ابزارهایی که نیاز به فایل دارند
  - DatePicker — تاریخ
- Placeholder
- Help Text
- اجباری/اختیاری
- محدودیت طول

**امکان Drag & Drop برای تغییر ترتیب فیلدها**

**پیش‌نمایش زنده:** همزمان با اضافه کردن فیلد، پیش‌نمایش فرم کاربر نهایی نمایش داده می‌شود

**اعتبارسنجی:** سیستم بررسی می‌کند که همه {variable_name} های پرامپت یک فیلد متناظر داشته باشند

### Apps/Showcase.cshtml — گالری نمونه خروجی

### Apps/Submit.cshtml — ارسال برای بررسی
- چک‌لیست پیش از ارسال:
  - آیا عنوان دارد؟ ✓
  - آیا توضیح کامل دارد؟ ✓
  - آیا حداقل یک فیلد ورودی دارد؟ ✓
  - آیا حداقل ۳ نمونه خروجی دارد؟ ✓
  - آیا پرامپت و فیلدها هماهنگ هستند؟ ✓
- دکمه "ارسال برای بررسی"
- پس از ارسال: Status = UnderReview

## نکات UX

- اگر سازنده برای اولین بار وارد می‌شود و هنوز ابزاری ندارد، یک onboarding wizard نمایش داده می‌شود
- وضعیت هر ابزار با Badge رنگی نمایش داده می‌شود:
  - Draft: خاکستری
  - UnderReview: آبی
  - Active: سبز
  - Suspended: قرمز
- هشدار: اگر ابزار بیش از ۳۰ روز در Draft باشد، یادآوری نمایش داده می‌شود
