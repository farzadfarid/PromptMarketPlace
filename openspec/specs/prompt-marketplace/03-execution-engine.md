# Execution Engine

## مهم‌ترین بخش فنی پروژه

این بخش مسئول اجرای امن پرامپت است. پرامپت اصلی هرگز به فرانت‌اند نمی‌رود.

---

## جریان کامل اجرا

```
کاربر فرم را پر می‌کند
         ↓
POST به سرور
         ↓
بررسی احراز هویت
         ↓
بررسی موجودی اعتبار
         ↓
Validate کردن ورودی‌ها
         ↓
کم کردن اعتبار از کیف‌پول
         ↓
رمزگشایی پرامپت (فقط روی سرور)
         ↓
جایگزینی متغیرها در پرامپت
         ↓
ارسال به OpenRouter
         ↓
پردازش خروجی بر اساس OutputType
         ↓
ذخیره نتیجه در دیتابیس
         ↓
توزیع درآمد به سازنده
         ↓
نمایش خروجی به کاربر
```

---

## مراحل جزئی

### مرحله ۱: اعتبارسنجی
- آیا کاربر لاگین است؟
- آیا ابزار فعال است (Status = Active)؟
- آیا کاربر صاحب ابزار نیست؟ (سازنده نمی‌تواند ابزار خودش را اجرا کند - جلوگیری از تقلب)
- آیا موجودی کافی است؟

### مرحله ۲: Validate ورودی‌ها
برای هر AppInputField بررسی شود:
- فیلد IsRequired پر شده باشد
- MaxLength رعایت شده باشد
- نوع فیلد با مقدار مطابقت داشته باشد

### مرحله ۳: ذخیره AppExecution با وضعیت Pending
قبل از هر چیز رکورد ذخیره می‌شود تا در صورت خطا قابل ردیابی باشد.

### مرحله ۴: کم کردن اعتبار
اعتبار کاربر کم می‌شود. اگر بعداً خطا بدهد، اعتبار بازمی‌گردد (Refund).

### مرحله ۵: رمزگشایی و جایگزینی
```
پرامپت رمزگشایی شده: "یک کمپین برای {brand_name} در حوزه {industry} بنویس..."
مقادیر ورودی: { brand_name: "دیجی‌کالا", industry: "فروشگاه آنلاین" }
پرامپت نهایی: "یک کمپین برای دیجی‌کالا در حوزه فروشگاه آنلاین بنویس..."
```

این عملیات فقط در حافظه سرور انجام می‌شود و هیچ‌جا ذخیره نمی‌شود.

### مرحله ۶: ارسال به OpenRouter
Request بر اساس OutputType متفاوت است:
- Text / Code: Chat Completion API
- Image: Image Generation API
- Video: Video Generation API (async)
- Audio: Audio Generation API

### مرحله ۷: پردازش خروجی بر اساس OutputType
(جزئیات در بخش بعد)

### مرحله ۸: توزیع درآمد
```
قیمت اعتبار کاربر = X ریال
سهم سازنده = X × CommissionPercent / 100
سهم پلتفرم = X × (100 - CommissionPercent) / 100
```
هر دو تراکنش در WalletTransaction ذخیره می‌شوند.

## خروجی‌های داینامیک

### OutputType: Text
- AI متن Markdown برمی‌گرداند
- در صفحه با یک Markdown Renderer نمایش داده می‌شود
- دکمه "کپی" و "دانلود txt" دارد

### OutputType: Code
- مشابه Text اما با Syntax Highlighter
- زبان برنامه‌نویسی در پرامپت مشخص می‌شود
- دکمه "کپی کد" دارد

### OutputType: Image
- URL تصویر از API برگشته می‌شود
- تصویر دانلود و در Storage سرور ذخیره می‌شود
- به کاربر با img tag نمایش داده می‌شود
- دکمه "دانلود" دارد

### OutputType: Video
- Video generation معمولاً async است
- سیستم یک Job ID می‌گیرد
- با Polling هر چند ثانیه وضعیت بررسی می‌شود
- پس از آماده شدن URL ذخیره می‌شود
- با video player نمایش داده می‌شود

### OutputType: Audio
- مشابه Image اما با audio player نمایش داده می‌شود

### OutputType: Form
- جالب‌ترین نوع خروجی
- AI یک JSON Schema برمی‌گرداند
- این Schema به یک فرم HTML تبدیل می‌شود
- مثال کاربرد: سازنده یک ابزار "تولید قرارداد" می‌سازد
  - کاربر اطلاعات اولیه را وارد می‌کند
  - AI یک فرم کامل‌تر با فیلدهای اختصاصی برمی‌گرداند
  - کاربر فرم را پر می‌کند
  - ارسال فرم، یک اجرای دوم را trigger می‌کند

## مدیریت خطا

| خطا | اقدام |
|-----|-------|
| API timeout | Status = Failed، اعتبار Refund می‌شود |
| خطای API | Status = Failed، اعتبار Refund می‌شود |
| ورودی نامعتبر | قبل از کم کردن اعتبار برگشت داده می‌شود |
| موجودی ناکافی | قبل از هر چیز خطا داده می‌شود |

---

## Rate Limiting و جلوگیری از سوءاستفاده

- هر کاربر حداکثر N اجرا در دقیقه (کانفیگ در SystemSetting)
- هر کاربر حداکثر M اجرا در روز
- سیستم Pattern Matching برای جلوگیری از reverse engineering پرامپت
  - اگر یک کاربر هزار بار همان ابزار را با ورودی‌های مشابه اجرا کند، Flag می‌شود
- سازنده می‌تواند ابزارش را برای IP خاص یا کشور خاص محدود کند

---

## امنیت پرامپت

- پرامپت با AES-256 رمزنگاری و در دیتابیس ذخیره می‌شود
- کلید رمزنگاری در Environment Variable است، نه دیتابیس
- پرامپت رمزگشایی شده فقط در حافظه RAM و فقط در مدت اجرا وجود دارد
- هیچ لاگی از محتوای پرامپت نوشته نمی‌شود
- خروجی کاربر هیچ اطلاعاتی از ساختار پرامپت فاش نمی‌کند
