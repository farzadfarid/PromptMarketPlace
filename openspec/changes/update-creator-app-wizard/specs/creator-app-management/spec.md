## MODIFIED Requirements

### Requirement: ساخت ابزار جدید — Wizard چند مرحله‌ای
سازنده ابزار را در ۳ مرحله می‌سازد. پس از تکمیل، به لیست ابزارها هدایت می‌شود.

**مرحله ۱: اطلاعات پایه**
- عنوان (اجباری، حداکثر ۲۰۰ کاراکتر)
- توضیح کوتاه (اجباری، حداکثر ۱۶۰ کاراکتر)
- توضیح کامل (Markdown، اجباری)
- دسته‌بندی (اجباری)
- تگ‌ها (اختیاری، `,` جداکننده)
- تصویر بندانگشتی (اختیاری، حداکثر ۵MB، فرمت‌های jpg/png/webp)

**مرحله ۲: تنظیمات AI**
- انتخاب نوع خروجی (OutputType): Text / Image / Video / Audio / Code / Form
- انتخاب مدل AI: لیست مدل‌ها بر اساس Capability فیلتر می‌شود
- هزینه اعتباری (پیش‌فرض بر اساس OutputType از Settings)

**مرحله ۳: فیلدها + پرامپت (ترکیبی)**
- دو ستون: سمت چپ Field Builder، سمت راست Prompt Editor
- سازنده ابتدا فیلدها را تعریف می‌کند، سپس پرامپت می‌نویسد

**Field Builder:**
- نام متغیر (فقط `a-z`, `0-9`, `_`، اجباری، یکتا)
- عنوان نمایشی (اجباری)
- نوع: Text / Textarea / Select / Number / Checkbox / DatePicker / FileUpload
- Placeholder (اختیاری)
- اجباری/اختیاری
- گزینه‌ها (فقط برای نوع Select، هر خط یک گزینه)
- جابجایی ترتیب (بالا/پایین)
- حذف

**Prompt Editor:**
- System Context (اختیاری) با راهنمای popover
- پرامپت اصلی (اجباری)
- Quick-picker: وقتی `{` در textarea تایپ می‌شود و حداقل یک فیلد تعریف‌شده وجود دارد، نوار انتخاب سریع ظاهر می‌شود
- درج متغیر: با کلیک روی فیلد در quick-picker یا کارت فیلد، `{name}` در موقعیت cursor درج می‌شود
- Variable chips: متغیرهای موجود در پرامپت نمایش داده می‌شوند — نارنجی اگر فیلد تعریف‌شده باشد، قرمز اگر نباشد

#### Scenario: سازنده فیلد اضافه می‌کند
- **WHEN** سازنده نام و عنوان فیلد را وارد کرده و Add را می‌زند
- **THEN** فیلد به لیست اضافه می‌شود و در Quick-picker ظاهر می‌شود

#### Scenario: Quick-picker نمایش داده می‌شود
- **WHEN** سازنده `{` را در prompt textarea تایپ می‌کند و حداقل یک فیلد تعریف‌شده وجود دارد
- **THEN** نوار Quick-picker با لیست فیلدهای تعریف‌شده ظاهر می‌شود

#### Scenario: درج متغیر در پرامپت
- **WHEN** سازنده روی یک فیلد در Quick-picker کلیک می‌کند
- **THEN** `{field_name}` در موقعیت cursor درج می‌شود و Quick-picker بسته می‌شود

#### Scenario: variable chip قرمز
- **WHEN** پرامپت حاوی `{variable}` است اما فیلدی با آن نام تعریف نشده
- **THEN** chip قرمز با آیکون ⚠ نمایش داده می‌شود

#### Scenario: ذخیره ابزار
- **WHEN** سازنده فرم را submit می‌کند
- **THEN** ابزار ذخیره می‌شود، فیلدها به عنوان `AppInputField` ثبت می‌شوند، و سازنده به لیست ابزارها هدایت می‌شود

## MODIFIED Requirements

### Requirement: ویرایش ابزار — Wizard یکپارچه
صفحه ویرایش SHALL از همان ساختار ۳ مرحله‌ای Create استفاده کند.

**قوانین قابلیت ویرایش:**
- ابزارهای Draft یا Suspended: همه بخش‌ها قابل ویرایش
- ابزارهای Active یا UnderReview: فقط اطلاعات پایه قابل ویرایش (پرامپت و تنظیمات AI قفل هستند)

**رفتار هنگام قفل بودن AI Settings:**
- کنترل‌های OutputType و AiModelId disabled نمایش داده می‌شوند
- مقادیر فعلی از طریق hidden input ارسال می‌شوند تا validation پاس شود
- System Context به صورت readonly نمایش داده می‌شود
- بخش Prompt Editor کاملاً مخفی می‌شود

#### Scenario: ویرایش ابزار Active
- **WHEN** سازنده ابزار Active را باز می‌کند
- **THEN** مرحله ۲ و بخش پرامپت مرحله ۳ قفل هستند اما فیلدها قابل مدیریتند
- **AND** تغییر اطلاعات پایه (مرحله ۱) ذخیره می‌شود بدون خطای validation

#### Scenario: sync فیلدها در ویرایش
- **WHEN** سازنده فرم ویرایش را submit می‌کند
- **THEN** فیلدهای قبلی حذف و فیلدهای جدید از FieldsJson جایگزین می‌شوند
