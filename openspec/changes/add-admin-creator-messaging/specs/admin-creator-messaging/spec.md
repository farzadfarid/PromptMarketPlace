## ADDED Requirements

### Requirement: مدل داده پیام‌رسانی
سیستم SHALL از دو موجودیت برای پیام‌رسانی استفاده کند:

**MessageThread:**
- یک مکالمه بین ادمین و یک سازنده مشخص
- AppId اختیاری است — thread می‌تواند درباره یک ابزار خاص یا موضوع عمومی باشد
- LastMessageAt برای مرتب‌سازی به‌روزرسانی می‌شود

**ThreadMessage:**
- هر پیام داخل یک thread
- `IsFromAdmin` تعیین می‌کند فرستنده ادمین بوده یا سازنده
- `IsRead` هنگامی که طرف مقابل thread را باز می‌کند true می‌شود

#### Scenario: ایجاد thread جدید
- **WHEN** ادمین از صفحه جزئیات ابزار پیام می‌فرستد
- **THEN** thread جدید با `AppId` ابزار و `CreatorProfileId` سازنده آن ابزار ایجاد می‌شود

#### Scenario: thread عمومی
- **WHEN** سازنده از inbox خود پیام جدید می‌فرستد
- **THEN** thread بدون AppId ایجاد می‌شود

---

### Requirement: پنل مدیریت پیام‌ها — ادمین
ادمین SHALL بتواند همه مکالمات با سازندگان را مشاهده و پاسخ دهد.

**صفحه لیست (`/Admin/Messages`):**
- همه thread ها مرتب بر اساس آخرین پیام (جدیدترین اول)
- نام سازنده، موضوع، پیش‌نمایش آخرین پیام
- اگر thread به ابزاری مرتبط است، نام ابزار به صورت tag نمایش داده می‌شود
- badge تعداد پیام‌های خوانده‌نشده روی هر thread
- thread های دارای پیام خوانده‌نشده با رنگ متفاوت highlight می‌شوند

**صفحه chat (`/Admin/Messages/{id}`):**
- پیام‌های ادمین: سمت چپ، حباب تیره
- پیام‌های سازنده: سمت راست، حباب نارنجی
- read receipt: tick دوگانه سبز اگر خوانده‌شده، tick ساده خاکستری اگر نه
- auto-scroll به آخرین پیام
- Reply form در پایین
- هنگام باز شدن، پیام‌های سازنده به عنوان خوانده‌شده علامت می‌خورند

#### Scenario: ادمین thread را باز می‌کند
- **WHEN** ادمین روی یک thread کلیک می‌کند
- **THEN** همه پیام‌های سازنده در آن thread به عنوان IsRead=true ثبت می‌شوند
- **AND** chat به آخرین پیام scroll می‌شود

#### Scenario: ادمین پاسخ می‌دهد
- **WHEN** ادمین پیام ارسال می‌کند
- **THEN** پیام با IsFromAdmin=true ذخیره می‌شود و LastMessageAt thread بروز می‌شود

#### Scenario: ادمین از صفحه ابزار پیام می‌فرستد
- **WHEN** ادمین در جزئیات ابزار روی "ارسال پیام" کلیک می‌کند
- **THEN** modal باز می‌شود، موضوع پیش‌پر با نام ابزار است
- **AND** پس از ارسال، thread جدید ایجاد شده و به صفحه chat آن thread هدایت می‌شود

### Requirement: پنل پیام‌ها — سازنده
سازنده SHALL بتواند مکالمات خود با ادمین را مشاهده، شروع کند، و پاسخ دهد.

**صفحه inbox (`/Creator/Messages`):**
- thread های متعلق به سازنده لاگین‌شده
- آخرین پیام پیش‌نمایش: ادمین یا شما پیشوند دارد
- badge خوانده‌نشده
- دکمه "پیام جدید" با modal: موضوع + اولین پیام

**صفحه chat (`/Creator/Messages/{id}`):**
- پیام‌های سازنده: سمت راست (نارنجی)
- پیام‌های ادمین: سمت چپ (بنفش/indigo)
- read receipt روی پیام‌های سازنده (آیا ادمین خوانده یا نه)
- دسترسی محدود: سازنده فقط thread های خودش را می‌بیند

#### Scenario: سازنده thread جدید می‌سازد
- **WHEN** سازنده موضوع و پیام اول را در modal وارد می‌کند و ارسال می‌کند
- **THEN** thread ساخته می‌شود، پیام اول با IsFromAdmin=false ثبت می‌شود، و به صفحه chat هدایت می‌شود

#### Scenario: سازنده thread دیگری را باز می‌کند
- **WHEN** سازنده ID یک thread که به او تعلق ندارد را درخواست می‌کند
- **THEN** Forbidden برگردانده می‌شود

#### Scenario: سازنده پاسخ می‌دهد
- **WHEN** سازنده پیام ارسال می‌کند
- **THEN** پیام با IsFromAdmin=false ذخیره می‌شود
- **AND** هنگام باز کردن thread توسط سازنده، پیام‌های ادمین به عنوان خوانده‌شده علامت می‌خورند

### Requirement: Unread Badge در Navigation
هر دو پنل ادمین و سازنده SHALL تعداد پیام‌های خوانده‌نشده را در سایدبار نمایش دهند.

- ادمین: تعداد پیام‌هایی که سازندگان فرستادند و ادمین نخوانده
- سازنده: تعداد پیام‌هایی که ادمین فرستاده و سازنده نخوانده
- badge فقط هنگامی نمایش داده می‌شود که تعداد > 0 باشد
- هر page load مقدار به‌روز می‌شود

#### Scenario: badge ادمین
- **WHEN** سازنده‌ای پیام جدیدی فرستاده که ادمین نخوانده
- **THEN** عدد badge در سایدبار ادمین نمایش داده می‌شود

#### Scenario: badge سازنده
- **WHEN** ادمین پیامی فرستاده که سازنده نخوانده
- **THEN** عدد badge در سایدبار سازنده نمایش داده می‌شود
